The [flow\_limits\_fix](@ref) parameter fixes the value of the [unit\_flow](@ref var_unit_flow) or [connection\_flow](@ref var_connection_flow) variable.

For units, If [operating\_points](@ref) is defined on a certain [unit\_\_to\_node](@ref) or [node\_\_to\_unit](@ref) flow, the corresponding [unit\_flow](@ref var_unit_flow) flow variable is decomposed into a number of sub-variables, [unit\_flow\_op](@ref var_unit_flow_op) one for each operating point, with an additional index, `i` to reference the specific operating point. [flow\_limits\_fix\_op](@ref) can thus be used to fix the value of one or more of the variables as desired.

> **Warning:** that this parameter should be set to 0 for history timeslices to avoid free energy in the model when using the [connection\_flow\_delay](@ref) parameter.
