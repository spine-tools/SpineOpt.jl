The definition of the [minimum\_operating\_point](@ref) parameter will trigger the creation of the [Constraint on minimum operating point](@ref constraint_minimum_operating_point). It sets a lower bound on the value of the [unit\_flow variable](@ref var_unit_flow) for a unit that is online.

It can be defined for [unit\_\_to\_node](@ref) or [node\_\_to\_unit](@ref) relationships, as well as their counterparts for node groups. It will then impose restrictions on the [unit\_flow variable](@ref var_unit_flow)s that indicate flows between the two members of the relationship for which the parameter is defined. The parameter is given as a fraction of the [capacity\_per\_unit](@ref) parameter. When the parameter is not included, the aforementioned constraint will not be created, which is equivalent to choosing a value of 0.
